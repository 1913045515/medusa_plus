## Why

当前虚拟产品"数字资料"类型的下载地址使用明文外部 URL，没有下载次数限制、过期机制和域名绑定校验，存在数据被无限制下载导致流量超额的风险，且无法防止直接链接传播。同时后台缺少集中的文件管理能力，导致文件上传、替换、停用分散在各产品配置中，运营效率低、难以审计。

## What Changes

- 在 admin 新增独立的"文件管理"模块，支持文件上传（仅上传到私有 S3 桶）、查看、删除与基础元数据管理（名称、描述、大小、类型）。
- 虚拟产品配置中"数字资料"类型放弃外部 URL 字段，改为关联文件管理模块中的一个文件记录。
- 在订单详情页（storefront），为已购买的数字资料虚拟产品展示受控下载入口，下载操作由后端生成 30 分钟有效的 S3 预签名 URL，通过本平台服务器中转，不直接暴露 S3 地址。
- 每个文件每位客户每自然日限制最多下载 3 次，超额后提示"今日下载次数已达上限，请明日再试"。
- 签名 URL 有效期为 30 分钟；若客户在超过 30 分钟后才点击下载，页面展示"下载链接已过期，请刷新页面以重新获取"，刷新后重新生成签名。

## Capabilities

### New Capabilities
- `file-asset-management`: admin 后台文件资产管理模块，支持上传到私有 S3、查看文件列表、删除文件，以及基础元数据管理。
- `virtual-product-file-binding`: 虚拟产品配置中，资料类型商品可以选择文件管理模块中已上传的文件，替代外部 URL。
- `secure-file-download`: 已购买数字资料商品的客户可在订单详情页通过 30 分钟有效签名 URL 下载文件，限每个文件每天最多 3 次，签名失效后提示刷新。

### Modified Capabilities
- `virtual-order-entitlements`: 订单行快照中的资料下载地址从外部 URL 升级为文件管理模块的文件引用（`file_asset_id`），下载入口改为通过后端受控签名接口访问。
- `virtual-product-catalog`: 虚拟产品资料配置字段从 `resource_download_url` 改为 `resource_file_asset_id`，关联文件管理模块记录。

## Non-goals

- 不支持将文件上传到本地磁盘或其他存储（仅 S3）。
- 不实现文件内容预览或在线查看（仅提供下载）。
- 不实现文件夹/分类管理层级（本次为平铺列表）。
- 不改造课程视频/封面媒体（已由 add-s3-course-media-storage 处理）。
- 不实现文件分享链接或公开访问模式。

## Impact

- Backend: `my-store` 新增文件资产模块、S3 上传/删除服务、下载次数限制表、admin API、store 下载 API。
- Admin: 新增文件管理路由与 UI；产品虚拟配置表单中资料字段改为文件选择器。
- Storefront: 订单详情页资料下载入口改为调用受控下载 API，处理签名过期与下载限额两种降级态。
- Database: 新增 `file_asset` 表、`file_download_log` 表（记录下载次数）；修改虚拟产品配置字段。
